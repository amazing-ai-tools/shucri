# UI/UX Design — shucri

Use this skill to design responsive, accessible frontend experiences.

## Stack
- React 18 + TypeScript, Vite 5
- lucide-react icons
- Plain CSS in `src/styles.css` (no design system yet — propose one before adding)

## Current UI
- `src/main.tsx` renders a single placeholder hero + a 3-card status grid. This is template copy meant to be replaced once the product is defined.

## Principles
- Mobile-first, responsive layouts.
- Accessibility is required: semantic HTML, keyboard navigation, ARIA where needed, visible focus, sufficient contrast.
- Every async surface needs loading, empty, and error states.
- Keep visual language consistent; don't introduce one-off components.
- Replace placeholder copy with real product copy as soon as direction exists (coordinate with Marketing / Legolas).
